# Tailwinter — структура проекта

## Правила размещения
1. `assets/` — только сырьё (модели, текстуры, звук). Никакой логики.
2. `game/actors/<actor>/` — актёры: сцена, скрипт, визуал, камера, анимации рядом.
3. `game/features/<domain>/<feature>/` — геймплейные фичи пакетами (скрипты/сцены/данные).
4. `game/props/<group>/<item>/` — интерактивные пропсы с их логикой/сценами.
5. `game/ui/` — весь UI: HUD, меню, темы. Логика UI рядом со сценами.
6. `game/vfx/` — визуальные эффекты (сцены, материалы, скрипты для FX).
7. `game/world/levels/<name>/` — уровни и специфические для уровня скрипты (terrain и др.).
8. `game/core/` — утилиты, конфиги, ввод, сохранения, общие для проекта.
9. `tools/` — редакторские инструменты, не используемые в рантайме.
10. `_temp/` — временное, что можно безопасно удалить; не коммитить постоянные данные.

## Пример: новый зверь
```
game/actors/creatures/boar/
  boar.tscn         # сцена существа
  boar.gd           # логика/AI
  anim/             # AnimationTree/клипы/бленды
  visual/           # если нужен wrapper под модель/материалы
```

## Пример: новая фича
```
game/features/crafting/workbench/
  workbench.gd
  workbench.tscn
  recipes.tres
```

## Naming
- snake_case, латиница, без пробелов.
- Сцена = имя папки/фичи (player.tscn, trap.tscn).
- Материалы/ресурсы рядом с фичей, если логические; «сырьё» остаётся в assets/.
